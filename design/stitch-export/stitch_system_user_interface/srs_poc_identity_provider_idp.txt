Software Requirements Specification (SRS): PoC Identity Provider (IdP)

1. Introduction
1.1 Purpose
This document describes all the features, requirements, and architecture for a Proof of Concept (PoC) Identity Provider (IdP).
What is an Identity Provider (IdP)? An Identity Provider is a system that manages user accounts and handles authentication (verifying who a user is). Think of it like the "Login with Google" or "Login with Facebook" system — Google and Facebook are acting as Identity Providers. Our PoC IdP is a simplified version of that concept.
What is a Proof of Concept (PoC)? A PoC is a small-scale, simplified version of a system built to demonstrate that an idea is technically feasible. It is not production-ready software; it is a working prototype.

1.2 Scope
The system provides three main capabilities:
Identity Management — End-users can register and log in.
OAuth 2.0 Authorization — External applications can securely authenticate users through our IdP.
Admin Dashboard — Administrators can manage client applications and user accounts.

1.3 Glossary of Key Terms
OAuth 2.0: An industry-standard protocol for a third-party application to access a user's data or act on their behalf.
JWT (JSON Web Token): A compact, self-contained token format used to securely transmit information.
Access Token: A credential that proves it has been authorized to access protected resources.
Authorization Code: A short-lived, single-use code that acts as an intermediary for an Access Token.
Client Application: Any external application (e.g., website or mobile app) that wants to use our IdP.
client_id: A unique, public identifier assigned to a Client Application.
client_secret: A confidential key assigned to a Client Application.
redirect_uri: The URL that the IdP will send the user back to after successful login.
Identity: An end-user account within the IdP system.
Identity Admin: An administrator who manages the IdP system through the Admin Dashboard.
